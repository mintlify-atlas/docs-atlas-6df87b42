## Overview

The CPU module is responsible for executing process instructions using a classic **fetch-decode-execute** cycle. It communicates with Memory to retrieve instructions and with Kernel through two separate interfaces: **Dispatch** and **Interrupt**.

### Architecture

The CPU operates with a dual-server architecture:

- **Dispatch Server**: Receives Process Control Blocks (PCBs) from Kernel and executes instructions
- **Interrupt Server**: Handles interruption requests from Kernel for process preemption

```c
// Initialize both servers
fd_cpu_dispatch = iniciar_servidor(logger_cpu, NULL, puerto_dispatch, "CPU DISPATCH");
fd_cpu_interrupt = iniciar_servidor(logger_cpu, NULL, puerto_interrupt, "CPU INTERRUPT");
```

## Fetch-Decode-Execute Cycle

### Fetch Phase

The CPU fetches instructions from Memory using the process's program counter (PC):

```c
bool fetchInstruccion(int fd, pcb* contexto, Instruccion *instruccion, t_log* logger) {
    Instruccion aux;
    send_fetch_instruccion(contexto->path, contexto->pc, fd);
    op_code cop = recibir_operacion(fd);
    
    if(cop != ENVIO_INSTRUCCION) {
        log_error(logger, "el cop no corresponde a una instruccion %d", cop);
        return false;
    }
    
    aux = recv_instruccion(fd);
    // Copy instruction data
    instruccion->opcode = malloc(sizeof(char) * strlen(aux.opcode) + 1);
    instruccion->operando1 = malloc(sizeof(char) * strlen(aux.operando1) + 1);
    instruccion->operando2 = malloc(sizeof(char) * strlen(aux.operando2) + 1);
    strcpy(instruccion->opcode, aux.opcode);
    strcpy(instruccion->operando1, aux.operando1);
    strcpy(instruccion->operando2, aux.operando2);
    
    log_info(logger, "PID: %d - FETCH - Program Counter: %d", contexto->pid, contexto->pc);
    return true;
}
```

### Decode Phase

For memory access instructions, the CPU translates logical addresses to physical addresses:

```c
DireccionFisica decodeInstruccion(Instruccion *instruccion, pcb* contexto, int fd_dispatch) {
    if (strcmp(instruccion->opcode, "MOV_IN") == 0 || 
        strcmp(instruccion->opcode, "F_READ") == 0 || 
        strcmp(instruccion->opcode, "F_WRITE") == 0 || 
        strcmp(instruccion->opcode, "MOV_OUT") == 0) {
        
        int direccion_logica = obtener_direccion_logica(instruccion);
        DireccionFisica direccion_fisica = traducir(direccion_logica, 
                                                      conexion_cpu_memoria, 
                                                      contexto->pid, 
                                                      fd_dispatch);
        if(direccion_fisica.marco >= 0) {
            return direccion_fisica;
        } else {
            direccion_fisica.marco = -1;
            return direccion_fisica;
        }
    } else {
        DireccionFisica direccion_fisica;
        direccion_fisica.marco = -2;
        return direccion_fisica;
    }
}
```

### Execute Phase

The CPU supports multiple instruction types:

- **Arithmetic**: `SET`, `SUM`, `SUB`
- **Memory Access**: `MOV_IN`, `MOV_OUT`
- **Synchronization**: `WAIT`, `SIGNAL`
- **File Operations**: `F_OPEN`, `F_CLOSE`, `F_SEEK`, `F_READ`, `F_WRITE`, `F_TRUNCATE`
- **Control Flow**: `JNZ`, `EXIT`, `SLEEP`

```c
void executeInstruccion(pcb* contexto_ejecucion, Instruccion instruccion, 
                        DireccionFisica direccion, int fd_dispatch, int fd_memoria) {
    log_info(logger_cpu, "PID: %d - Ejecutando: %s - %s %s", 
             contexto_ejecucion->pid, instruccion.opcode, 
             instruccion.operando1, instruccion.operando2);

    if (strcmp(instruccion.opcode, "SET") == 0) {
        setInstruccion(contexto_ejecucion, instruccion, logger_cpu);
    } else if (strcmp(instruccion.opcode, "SUM") == 0) {
        sumInstruccion(contexto_ejecucion, instruccion, logger_cpu);
    } else if (strcmp(instruccion.opcode, "SUB") == 0) {
        subInstruccion(contexto_ejecucion, instruccion, logger_cpu);
    } else if (strcmp(instruccion.opcode, "SLEEP") == 0) {
        ya_desalojo = true;
        flag_ciclo = false;
        sleepInstruccion(contexto_ejecucion, instruccion, logger_cpu, fd_dispatch);

}
```

## Dispatch Server

The Dispatch server receives PCBs from Kernel and executes them:

```c
static void procesar_conexion_dispatch(void* void_args) {
    int *args = (int*) void_args;
    int cliente_socket_dispatch = *args;

    op_code cop;
    while (cliente_socket_dispatch != -1 && !recibio_interrupcion) {
        if (recv(cliente_socket_dispatch, &cop, sizeof(op_code), 0) != sizeof(op_code)) {
            log_info(logger_cpu, "El cliente se desconecto de DISPATCH");
            return;
        }
        
        switch (cop) {
            case ENVIO_PCB: 
                flag_ciclo = true;
                ya_desalojo = false;
                contexto = recv_pcb(cliente_socket_dispatch);
                
                if (contexto->pid != -1) {
                    ciclo_instruccion(contexto, cliente_socket_dispatch, 
                                     cliente_socket_dispatch, logger_cpu);
                    pcb_destroyer(contexto);
                    break;    
                } else {
                    log_error(logger_cpu, "Error al recibir el PCB");
                }
                break;
            // ... other cases
        }
    }
}
```

## Interrupt Server

The Interrupt server handles two types of interruptions:

- **Standard Interruption**: For quantum expiration (Round Robin)
- **Finalize Interruption**: To terminate a process immediately

```c
static void procesar_conexion_interrupt(void* void_args) {
    int *args = (int*) void_args;
    int cliente_socket_interrupt = *args;

    op_code cop;
    while (cliente_socket_interrupt != -1) {
        if (recv(cliente_socket_interrupt, &cop, sizeof(op_code), 0) != sizeof(op_code)) {
            log_info(logger_cpu, "El cliente se desconecto de INTERRUPT");
            return;
        }
        
        switch (cop) {
            case INTERRUPCION:
                int pid_recibido = recv_interrupcion(cliente_socket_interrupt);
                if(contexto->pid == pid_recibido) {
                    recibio_interrupcion = true;
                } 
                break;
                
            case INTERRUPCION_FINALIZAR:
                int pid_recibido_finalizar = recv_interrupcion(cliente_socket_interrupt);
                log_info(logger_cpu, "se finaliza el proceso %d, mientras ejecutaba el proceso %d",
                         pid_recibido_finalizar, contexto->pid);
                if(contexto->pid == pid_recibido_finalizar) {
                    recibio_interrupcion_finalizar = true;
                }
                break;
        }
    }
}
```

## Context Switching

The CPU handles context switching through interruptions:

```c
void ciclo_instruccion(pcb* contexto, int cliente_socket_dispatch, 
                       int cliente_socket_interrupt, t_log* logger) {
    while (cliente_socket_dispatch != -1 && !recibio_interrupcion && 
           !recibio_interrupcion_finalizar && flag_ciclo) {
        
        Instruccion * instruccion = malloc(sizeof(Instruccion));
        DireccionFisica direccion;
        
        if(fetchInstruccion(conexion_cpu_memoria, contexto, instruccion, logger)) {
            direccion = decodeInstruccion(instruccion, contexto, cliente_socket_dispatch);
            
            if(direccion.marco >= 0 || direccion.marco == -2) {
                contexto->pc++;
                executeInstruccion(contexto, *instruccion, direccion, 
                                  cliente_socket_dispatch, conexion_cpu_memoria);
            } else {
                // Page fault handling
                free(instruccion->opcode);
                free(instruccion->operando1);
                free(instruccion->operando2);
                free(instruccion);
                return;
            }
        }
        free(instruccion->opcode);
        free(instruccion->operando1);
        free(instruccion->operando2);
        free(instruccion);
    }

    if(recibio_interrupcion) {
        recibio_interrupcion = false;
        if (!ya_desalojo) {
            send_pcbDesalojado(contexto, "INTERRUPCION", "", 
                             cliente_socket_dispatch, logger);
        }
        return;
    } else if (recibio_interrupcion_finalizar) {
        recibio_interrupcion_finalizar = false;
        return;
    }
}
```

## PCB Handling

The CPU maintains the process context (PCB) during execution, updating:

- **Program Counter (PC)**: Incremented after each instruction
- **CPU Registers**: Modified by arithmetic and memory operations
- **Process State**: Updated when returning to Kernel

## Thread Architecture

The CPU uses a multi-threaded architecture:

- **Main Thread**: Listens for connections
- **Dispatch Thread**: Handles process execution
- **Interrupt Thread**: Handles interruption requests

```c
int server_escuchar(int fd_cpu_interrupt, int fd_cpu_dispatch) {
    int socket_cliente_interrupt = esperar_cliente(logger_cpu, "CPU INTERRUPT", 
                                                    fd_cpu_interrupt);
    int socket_cliente_dispatch = esperar_cliente(logger_cpu, "CPU DISPATCH", 
                                                   fd_cpu_dispatch);
    
    if (socket_cliente_interrupt != -1 && socket_cliente_dispatch != -1) {
        // Thread for dispatch server
        pthread_t hilo_dispatch;
        pthread_create(&hilo_dispatch, NULL, (void*) procesar_conexion_dispatch, 
                      (void*) &socket_cliente_dispatch);
        pthread_detach(hilo_dispatch);
        
        // Thread for interrupt server
        pthread_t hilo_interrupt;
        pthread_create(&hilo_interrupt, NULL, (void*) procesar_conexion_interrupt, 
                      (void*) &socket_cliente_interrupt);
        pthread_detach(hilo_interrupt);
        
        return 1;
    }
    return 0;
}
```
