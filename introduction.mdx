# Welcome to MAPPA OS Simulator

MAPPA OS Simulator is a comprehensive operating system implementation that simulates the core components of a modern OS. Built in C, it provides hands-on experience with operating system concepts including process scheduling, memory management, file systems, and inter-process communication.

## What is MAPPA OS Simulator?

MAPPA OS Simulator is an educational operating system that implements:

- **CPU Module**: Executes processes using a fetch-decode-execute cycle with support for arithmetic, memory access, I/O, and file operations
- **Memory Module**: Manages virtual memory with paging, TLB, and page replacement algorithms (FIFO and LRU)
- **Kernel Module**: Coordinates process scheduling with multiple algorithms (FIFO, Round Robin, Priorities) and manages system resources
- **Filesystem Module**: Provides a FAT-based file system with block allocation and swap space for virtual memory

## System Architecture

MAPPA OS consists of four independent modules that communicate via TCP sockets:

```
┌─────────────┐     ┌─────────────┐
│   Kernel    │────▶│     CPU     │
│             │◀────│ (Dispatch + │
│             │     │  Interrupt) │
└──────┬──────┘     └──────┬──────┘
       │                   │
       │                   │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│  Filesystem │────▶│   Memory    │
│             │◀────│             │
└─────────────┘     └─────────────┘
```

- **Kernel**: Central coordinator managing process lifecycle, scheduling, resources, and file operations
- **CPU**: Executes process instructions with dual servers (dispatch for execution, interrupt for preemption)
- **Memory**: Manages page tables, handles page faults, stores instructions, and provides paging support
- **Filesystem**: Maintains FAT, manages blocks, and provides swap space for virtual memory

## Instruction Set

The CPU supports a rich instruction set:

- **Arithmetic**: `SET`, `SUM`, `SUB`
- **Memory Access**: `MOV_IN`, `MOV_OUT`
- **Control Flow**: `JNZ`, `EXIT`
- **Synchronization**: `WAIT`, `SIGNAL`, `SLEEP`
- **File Operations**: `F_OPEN`, `F_CLOSE`, `F_SEEK`, `F_READ`, `F_WRITE`, `F_TRUNCATE`

See the [CPU Instructions](/api/cpu/instructions) reference for complete details.

## Scheduling Algorithms

The kernel supports three process scheduling algorithms:

<Accordion title="FIFO (First In, First Out)">
Non-preemptive scheduling where processes execute in arrival order until completion or blocking.
</Accordion>

<Accordion title="Round Robin">
Preemptive scheduling with configurable quantum. Each process executes for a time slice before being preempted.
</Accordion>

<Accordion title="Priorities">
Preemptive priority-based scheduling where lower priority numbers execute first. Higher priority processes can preempt lower priority ones.
</Accordion>

## Page Replacement Algorithms

The memory module implements two page replacement algorithms:

<Tabs>
  <Tab title="FIFO">
    **First In, First Out** - Replaces the oldest page in memory. Simple but may not be optimal for all workloads.
  </Tab>
  <Tab title="LRU">
    **Least Recently Used** - Replaces the page that hasn't been accessed for the longest time. Better performance for temporal locality.
  </Tab>
</Tabs>
