title: 'Protocol API'
description: 'Inter-process communication protocol for MAPPA OS modules'

The Protocol API provides functions for serializing, sending, and receiving messages between different modules of the MAPPA OS simulator (Kernel, Memory, CPU, and Filesystem).

## Data Structures

Enumeration of all operation codes used in the protocol.

```c
typedef enum {
    // System Operations
    HAY_ALGO_MAL,
    MENSAJE,
    PAQUETE,

    PCB_SIGNAL,
    PCB_WAIT,
    PCB_SLEEP,
    PCB_INTERRUPCION,
    PCB_EXIT,
    PCB_PAGEFAULT,
    
    // Process Management
    INICIALIZAR_PROCESO,
    FINALIZAR_PROCESO,
    CAMBIAR_ESTADO,
    INTERRUPCION,
    INTERRUPCION_FINALIZAR,
    
    // Resource Management
    MANEJAR_IO,
    MANEJAR_WAIT,
    MANEJAR_SIGNAL,
    
    // Memory Operations
    CONEXION_MEMORIA,
    PEDIDO_MARCO,
    ENVIO_MARCO,
    TAMANIO_PAGINA,
    CARGAR_PAGINA,
    PAGINA_CARGADA,
    
    // Swap Operations
    RESERVA_SWAP,
    BLOQUES_RESERVADOS,
    LIBERACION_SWAP,
    PEDIDO_SWAP,
    ESCRIBIR_SWAP,
    
    // Filesystem Operations
    ABRIR_ARCHIVO,

    PEDIDO_ESCRITURA_FS,
    FIN_ESCRITURA,

    ARCHIVO_NO_EXISTE,
```

### t_buffer

Buffer structure for storing serialized data.

```c
typedef struct {
    int size;        // Size of the data in bytes
    void *stream;    // Pointer to the data
} t_buffer;
```

### t_paquete

Packet structure for sending data with an operation code.

```c
typedef struct {
    op_code codigo_operacion;  // Operation code identifying the message type
    t_buffer *buffer;          // Buffer containing the serialized data
} t_paquete;
```

### estado_proceso

Process state enumeration.

```c
typedef enum {
    NEW,
    READY,
    EXEC,
    BLOCK,
    EXIT_ESTADO
} estado_proceso;
```

### pcb

Process Control Block structure.

```c
typedef struct {
    int pid;                    // Process identifier
    int pc;                     // Program counter (next instruction to execute)
    int size;                   // Process size
    int prioridad;              // Process priority
    int tiempo_ejecucion;       // Execution time
    char *path;                 // Path to the process instructions file
    estado_proceso estado;      // Current process state
    t_registros *registros;     // CPU registers
    t_list *archivos;           // List of open files
} pcb;
```

### Instruccion

Instruction structure.

```c
typedef struct {
    char *opcode;      // Operation code (e.g., "SET", "ADD", "MOV_IN")
    char *operando1;   // First operand
    char *operando2;   // Second operand
} Instruccion;
```

## Core Functions

### serializar_paquete

Serializes a packet into a byte stream for transmission.

```c
void* serializar_paquete(t_paquete* paquete, int bytes);
```

**Parameters:**
- `paquete`: Pointer to the packet to serialize
- `bytes`: Total size in bytes (typically `paquete->buffer->size + 2*sizeof(int)`)

**Returns:**
- Pointer to the serialized data (must be freed by caller)

**Description:**
Serializes a packet by copying the operation code, buffer size, and buffer data into a contiguous memory block. The format is:
1. Operation code (4 bytes)
2. Buffer size (4 bytes)
3. Buffer data (variable size)

**Example:**
```c
t_paquete* paquete = crear_paquete(ENVIO_PCB);
agregar_a_paquete(paquete, &pid, sizeof(int));
int bytes = paquete->buffer->size + 2*sizeof(int);
void* serializado = serializar_paquete(paquete, bytes);
send(socket, serializado, bytes, 0);
free(serializado);
```

### enviar_mensaje

Sends a simple text message to a socket.

```c
void enviar_mensaje(char* mensaje, int socket_cliente);
```

**Parameters:**
- `mensaje`: Null-terminated string to send
- `socket_cliente`: Socket file descriptor

**Description:**
Creates a packet with operation code `MENSAJE`, serializes the message string, and sends it through the socket.

**Example:**
```c
enviar_mensaje("Process initialized", socket_kernel);
```

### recibir_operacion

Receives the operation code from a socket.

```c
int recibir_operacion(int socket_cliente);
```

**Parameters:**
- `socket_cliente`: Socket file descriptor

**Returns:**
- Operation code as an integer
- `-1` if the connection was closed or an error occurred

**Description:**
Reads the first 4 bytes from the socket to determine the operation code. This should be called before receiving the actual data payload.

**Example:**
```c
int cod_op = recibir_operacion(socket_cliente);
if (cod_op == -1) {
    log_error(logger, "Connection closed");
    return;
}

switch(cod_op) {
    case ENVIO_PCB:
        pcb* proceso = recv_pcb(socket_cliente);
        break;
    case MENSAJE:
        recibir_mensaje(logger, socket_cliente);
        break;
}
```

### recibir_buffer

Receives a buffer from a socket.

```c
void* recibir_buffer(int* size, int socket_cliente);
```

**Parameters:**
- `size`: Pointer to store the received buffer size
- `socket_cliente`: Socket file descriptor

**Returns:**
- Pointer to the received buffer (must be freed by caller)

**Description:**
First receives the buffer size (4 bytes), then allocates memory and receives the buffer data.

### recibir_mensaje

Receives and logs a text message from a socket.

```c
void recibir_mensaje(t_log* logger, int socket_cliente);
```

**Parameters:**
- `logger`: Logger instance for output
- `socket_cliente`: Socket file descriptor

**Description:**
Receives a text message and logs it using the provided logger. Automatically frees the received buffer.

### recibir_paquete

Receives a packet and deserializes it into a list.

```c
t_list* recibir_paquete(int socket_cliente);
```

**Parameters:**
- `socket_cliente`: Socket file descriptor

**Returns:**
- List containing the deserialized packet elements
- `NULL` if an error occurred

**Description:**
Receives a buffer and unpacks it into a list. Each element in the packet is stored with its size prefix. The returned list must be destroyed when no longer needed.

**Example:**
```c
t_list* paquete = recibir_paquete(socket_cliente);
if (paquete == NULL) {
    log_error(logger, "Error receiving packet");
    return;
}

int* valor1 = list_get(paquete, 0);
char* valor2 = list_get(paquete, 1);

list_destroy_and_destroy_elements(paquete, free);
```

## Packet Management

### crear_paquete

Creates a new packet with the specified operation code.

```c
t_paquete* crear_paquete(op_code codigo_op);
```

**Parameters:**
- `codigo_op`: Operation code for the packet

**Returns:**
- Pointer to the newly created packet

**Example:**
```c
t_paquete* paquete = crear_paquete(ENVIO_PCB);
```

### agregar_a_paquete

Adds data to a packet.

```c
void agregar_a_paquete(t_paquete* paquete, void* valor, int tamanio);
```

**Parameters:**
- `paquete`: Packet to add data to
- `valor`: Pointer to the data to add
- `tamanio`: Size of the data in bytes

**Description:**
Appends data to the packet's buffer. Each value is stored with a size prefix for deserialization.

**Example:**
```c
t_paquete* paquete = crear_paquete(INICIALIZAR_PROCESO);
int pid = 42;
char* path = "/path/to/instructions";

agregar_a_paquete(paquete, &pid, sizeof(int));
agregar_a_paquete(paquete, path, strlen(path) + 1);
```

### enviar_paquete

Serializes and sends a packet through a socket.

```c
void enviar_paquete(t_paquete* paquete, int socket_cliente);
```

**Parameters:**
- `paquete`: Packet to send
- `socket_cliente`: Socket file descriptor

**Description:**
Serializes the packet and sends it. Does not free the packet.

### eliminar_paquete

Frees a packet and its buffer.

```c
void eliminar_paquete(t_paquete* paquete);
```

**Parameters:**
- `paquete`: Packet to free

## PCB Operations

### send_pcb

Sends a Process Control Block to a module.

```c
void send_pcb(pcb* contexto, int fd_modulo);
```

**Parameters:**
- `contexto`: PCB to send
- `fd_modulo`: Socket file descriptor of the destination module

**Example:**
```c
pcb* proceso = crear_pcb(pid, path, prioridad);
send_pcb(proceso, socket_cpu);
```

### recv_pcb

Receives a Process Control Block from a module.

```c
pcb* recv_pcb(int fd_modulo);
```

**Parameters:**
- `fd_modulo`: Socket file descriptor

**Returns:**
- Pointer to the received PCB (must be freed with `pcb_destroyer`)

**Example:**
```c
pcb* proceso = recv_pcb(socket_kernel);
log_info(logger, "Received PCB for PID: %d", proceso->pid);
```

### pcb_destroyer

Frees a PCB and all its associated resources.

```c
void pcb_destroyer(pcb* contexto);
```

**Parameters:**
- `contexto`: PCB to free

## Usage Example

Here's a complete example of sending a process from Kernel to CPU:

```c
// Create a PCB
pcb* proceso = malloc(sizeof(pcb));
proceso->pid = 1;
proceso->pc = 0;
proceso->prioridad = 5;
proceso->path = strdup("/path/to/instructions");
proceso->estado = EXEC;

// Initialize registers
proceso->registros = malloc(sizeof(t_registros));
proceso->registros->ax = malloc(sizeof(uint32_t));
*proceso->registros->ax = 0;
// ... initialize other registers

// Initialize file list
proceso->archivos = list_create();

// Send to CPU
send_pcb(proceso, socket_cpu);
log_info(logger, "PCB sent to CPU for execution");

// Receive evicted PCB
int cod_op = recibir_operacion(socket_cpu);
if (cod_op == PCB_INTERRUPCION) {
    pcb* pcb_evicted;
    char* motivo;
    recv_pcbDesalojado(socket_cpu, &pcb_evicted, &motivo);
    log_info(logger, "Process %d interrupted: %s", pcb_evicted->pid, motivo);
    
    // Clean up
    free(motivo);
    pcb_destroyer(pcb_evicted);
}
```

## Notes

- All send/recv functions handle serialization and deserialization automatically
- Memory management: Received packets, PCBs, and strings must be freed by the caller
- The protocol uses size prefixes for all variable-length data
- Operation codes determine the message format and handler
- Always call `recibir_operacion` before receiving the payload to determine the message type
