## Overview

The paging API provides functions for managing virtual memory through page tables, translating logical addresses to physical frames, and handling page faults. The system uses a per-process page table structure stored in `lista_tablas_de_procesos`.

## Data Structures

### entrada_pagina

Represents a single page table entry for a process.

```c
typedef struct {
    int num_marco;           // Physical frame number (-1 if not in memory)
    int modificado;          // Modified bit (1 if page has been written to)
    int en_memoria;          // Presence bit (1 if page is in physical memory)
    int pid;                 // Process ID that owns this page
    int num_pagina;          // Logical page number
    time_t ultimo_tiempo_uso; // Last access time (for LRU algorithm)
    int posicion_swap;       // Position in swap space
} entrada_pagina;
```

**Fields:**
- `num_marco`: Physical frame number where the page is loaded, or -1 if not in memory
- `modificado`: Dirty bit indicating if the page has been modified (requires write-back to swap)
- `en_memoria`: Presence bit indicating if the page is currently in physical memory
- `pid`: Process ID that owns this page
- `num_pagina`: Logical page number within the process's address space
- `ultimo_tiempo_uso`: Timestamp of last access, used by the LRU replacement algorithm
- `posicion_swap`: Block number in swap space where this page is stored

## Process Initialization

### inicializar_proceso

Initializes a page table for a new process.

```c
t_list* inicializar_proceso(pcb* proceso, t_list* bloques_reservados)
```

**Parameters:**
- `proceso`: Pointer to the process control block containing PID and size
- `bloques_reservados`: List of swap block numbers reserved for this process

**Returns:**
- `t_list*`: Pointer to the newly created page table (list of `entrada_pagina` structures)

**Description:**

Creates a page table for a new process with all pages initially marked as not in memory. Each page table entry is initialized with:
- `num_marco = -1` (not in physical memory)
- `modificado = 0` (not modified)
- `en_memoria = 0` (not present)
- `pid` set to the process ID
- `num_pagina` set to the page index
- `ultimo_tiempo_uso = NULL` (no access yet)
- `posicion_swap` set from the reserved swap blocks

The function logs the process creation: `PID: <pid> - Tamaño: <num_pages>`

**Example usage:**
```c
// From main.c:81-92
pcb* proceso = recv_pcb(cliente_socket);
log_info(logger_memoria, "Creación de Proceso PID: %d, path: %s", proceso->pid, proceso->path);
int cant_paginas_necesarias = paginas_necesarias(proceso);
send_reserva_swap(conexion_memoria_filesystem, cant_paginas_necesarias);
t_list* bloques_reservados = recv_reserva_swap(conexion_memoria_filesystem);
t_list* tabla_paginas = inicializar_proceso(proceso, bloques_reservados);
list_add(lista_tablas_de_procesos, tabla_paginas);
```

## Address Translation

### obtener_nro_marco_memoria

Translates a logical page number to a physical frame number.

```c
int obtener_nro_marco_memoria(int num_pagina, int pid_actual)
```

**Parameters:**
- `num_pagina`: Logical page number to translate
- `pid_actual`: Process ID requesting the translation

**Returns:**
- `int`: Physical frame number if page is in memory, -1 if page fault

**Description:**

Searches through `lista_tablas_de_procesos` to find the page table for the specified process, then looks up the page entry. If the page is in memory (`en_memoria == 1`), returns the frame number and logs the translation. Otherwise, returns -1 to indicate a page fault.

**Log output:**
```
PID: <pid> - Pagina: <num_pagina> - Marco: <num_marco>
```

**Example usage:**
```c
// From main.c:103-118
int pid;
int numero_pagina;
recv_pedido_marco(cliente_socket, &pid, &numero_pagina);
int numeroMarco = obtener_nro_marco_memoria(numero_pagina, pid);
if(numeroMarco >= 0){
    send_marco(cliente_socket, numeroMarco);
}else{
    // Send page fault notification
    t_paquete* paquetePageFault = crear_paquete(PCB_PAGEFAULT);
    agregar_a_paquete(paquetePageFault, &numeroMarco, sizeof(int));
    enviar_paquete(paquetePageFault, cliente_socket);
    eliminar_paquete(paquetePageFault);
}
```

**Source:** memoria/src/main.c:299-313

### buscar_tabla_pagina

Finds the page table for a specific process.

```c

```

**Parameters:**
- `pid_actual`: Process ID to search for

**Returns:**
- `t_list*`: Pointer to the page table (list of `entrada_pagina`), or NULL if not found

**Description:**

## Page Fault Handling

### tratar_page_fault

Handles a page fault by loading a page from swap into physical memory.

```c
int tratar_page_fault(int num_pagina, int pid_actual)
```

**Parameters:**
- `num_pagina`: Logical page number that caused the fault
- `pid_actual`: Process ID that owns the page

**Returns:**
- `int`: Physical frame number where the page was loaded

**Description:**

Handles a page fault through the following steps:

1. **Locate page entry**: Finds the page table and gets the faulting page entry
2. **Log page fault**: Logs "PAGE FAULT" in lime color
3. **Load from swap**: Requests the page content from filesystem using `send_pedido_swap()` and receives it with `recv_leido_swap()`
4. **Find/allocate frame**:
   - Calls `buscar_marco_libre()` to find an empty frame
   - If no free frames exist (`nro_marco == -1`), calls `usar_algoritmo()` to select a victim page for replacement
5. **Write page to memory**: Calls `escribir_bloque_en_memoria()` to copy the page content into the frame
6. **Update page entry**:
   - Sets `num_marco` to the allocated frame
   - Sets `en_memoria = 1`
   - Sets `ultimo_tiempo_uso = time(NULL)`
7. **Update tracking**:
   - Adds the page to `paginas_en_memoria` list
   - Marks the frame as occupied in `bitarray_marcos_ocupados`

**Example usage:**
```c
// From main.c:187-196
int pid_pag;
int pagina_a_cargar;
recv_cargar_pagina(cliente_socket, &pid_pag, &pagina_a_cargar);
int numero_marco = tratar_page_fault(pagina_a_cargar, pid_pag);
send_pagina_cargada(cliente_socket);
```

**Source:** memoria/src/main.c:506-558

## Process Termination

### terminar_proceso

Cleans up all resources for a terminated process.

```c
void terminar_proceso(int pid)
```

**Parameters:**
- `pid`: Process ID to terminate

**Description:**

Performs cleanup when a process terminates:
1. Calls `eliminar_espacio_memoria()` to free all physical frames used by the process
2. Gets the page table and collects all swap block positions
3. Calls `eliminar_tabla_paginas()` to destroy the page table
4. Sends `send_liberacion_swap()` to filesystem to free swap space

**Source:** memoria/src/main.c:409-421

### eliminar_espacio_memoria

Frees all physical frames used by a process.

```c
void eliminar_espacio_memoria(int pid)
```

**Parameters:**
- `pid`: Process ID whose frames should be freed

**Description:**

Iterates through `paginas_en_memoria` and removes all entries belonging to the specified process, marking their frames as free in `bitarray_marcos_ocupados`.

**Source:** memoria/src/main.c:427-439

### eliminar_tabla_paginas

Destroys a process's page table.

```c
void eliminar_tabla_paginas(int pid)
```

**Parameters:**
- `pid`: Process ID whose page table should be destroyed

**Description:**

Finds and removes the page table for the specified process from `lista_tablas_de_procesos`, freeing all page entries.

**Source:** memoria/src/main.c:441-458

## Thread Safety

All memory operations use mutexes for thread safety:
- `mx_memoria`: Protects physical memory reads/writes
- `mx_bitarray_marcos_ocupados`: Protects frame allocation bitmap
- `mx_paginas_en_memoria`: Protects the list of loaded pages
